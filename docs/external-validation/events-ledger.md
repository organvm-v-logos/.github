## Hosting playbook (lowest-friction path)

The cheapest event to host with a verifiable record is a **GitHub Discussions
office-hours / AMA thread**:

1. Enable Discussions on a public org repo (e.g. `public-process`) if not already.
2. Open a pinned thread: *"Open office hours — ask me anything about building an
   eight-organ system in public."* State a window (e.g. a 48-hour async AMA, or a
   live 1-hour slot) and a clear invitation.
3. Announce it: org profile, essay footers, RSS, any social channel.
4. Engage every external participant who shows up.
5. After it closes, write a short recap and append a row above linking the
   thread. The thread itself is the durable record.

Other eligible formats: a live essay discussion / reading group, a recorded
stream or Space, a workshop. Any of them count as long as a non-creator
participates and a record survives.

> **Why async-first:** it removes timezone friction, maximizes the chance a real
> external participant shows up, and produces a permanent, linkable artifact for
> verification.
